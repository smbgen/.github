# Contributing to smbgen

Thanks for your interest in contributing.

## Before You Start

- Search existing issues before opening a new one.
- Keep changes focused and scoped.
- Prefer opening a discussion for broad design questions before implementation.

## Development Workflow

1. Fork the repository.
2. Create a feature branch.
3. Make changes with clear commit messages.
4. Add or update tests when behavior changes.
5. Open a pull request using the provided template.

## Pull Request Expectations

- Explain what changed and why.
- Link related issues.
- Include screenshots or logs for UI or behavioral changes.
- Keep PRs reviewable in size.

## Code Style

- Follow existing project conventions.
- Favor readability and maintainability over cleverness.
- Update documentation for user-facing changes.

## Community

By participating, you agree to follow our Code of Conduct in CODE_OF_CONDUCT.md.
