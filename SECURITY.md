# Security Policy

## Supported Versions

Security support is provided for actively maintained repositories and release lines.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Instead, report privately by contacting the organization maintainers through your designated security channel.

When reporting, include:

- affected repository and version
- impact and severity assessment
- reproduction steps or proof of concept
- any known mitigations

## Response Process

- We will acknowledge receipt as quickly as possible.
- We will investigate and validate the report.
- We will coordinate remediation and responsible disclosure timing.

Thanks for helping keep the ecosystem secure.
